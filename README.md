# AI-Project — Safety object detection (YOLOv8)

This repository contains a simple pipeline for building a custom workplace safety object detection dataset, auto-labeling it with Grounding DINO, cleaning the labels, converting the data to YOLO format, and training a YOLOv8 model.

Key capabilities:
- Auto-generate YOLO labels from class-specific text prompts
- Preview and clean image/label pairs before training
- Split valid examples into `train` / `val` / `test`
- Train and evaluate a YOLOv8 model with Ultralytics
- Utility scripts for downloading, renaming, HEIC conversion, and video frame extraction

---

## What it detects

The default configuration supports **four classes** (indices `0`–`3`):

| Index | Class               |
|-------|---------------------|
| 0     | `safety_helmet`     |
| 1     | `fire_extinguisher` |
| 2     | `emergency_exit`    |
| 3     | `traffic_cone`      |

These class names and indices are shared across `auto_label.py`, `split_dataset.py`, `clean_dataset.py`, `preview_labels.py`, and `yolo_dataset/data.yaml`.

## Quick start

This project requires **Python 3.11** and should be run inside a Python 3.11 virtual environment.

```bash
python3.11 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

1. Add raw images into `dataset/<class_name>/`.
2. Run `python auto_label.py` to generate YOLO label files in `labels/`.
3. Run `python preview_labels.py` to inspect the auto-labeled outputs.
4. Optionally clean bad examples with `python clean_dataset.py`.
5. Run `python split_dataset.py` to build `yolo_dataset/` and `yolo_dataset/data.yaml`.
6. Train and evaluate with `python train.py`.

## Pipeline overview

### 1. Gather images

Place images inside `dataset/<class_name>/` for each target class. Use scripts under `useful scripts/` to collect or prepare images if needed.

### 2. Auto-label with Grounding DINO

Run `python auto_label.py`.

- Uses Grounding DINO text prompts per class
- Writes normalized YOLO labels in `labels/<class>/`
- Creates empty `.txt` files when no detection is found
- Ensure `CLASS_INDICES` matches your label indexing and `data.yaml`

### 3. Preview labels

Run `python preview_labels.py`.

- Generates sample images with drawn boxes
- Saves outputs to `label_preview/`
- Helps catch missing or incorrect detections early

### 4. Clean the dataset

Run `python clean_dataset.py`.

- Opens an OpenCV review interface
- Allows keeping or deleting image/label pairs
- Supports optional class filtering: `python clean_dataset.py <class_name>`

### 5. Split data for YOLO

Run `python split_dataset.py`.

- Copies only examples with non-empty labels
- Writes `yolo_dataset/data.yaml`
- Produces a seeded `train` / `val` / `test` split

### 6. Train and evaluate

Run `python train.py`.

- Trains a YOLOv8 model if needed
- Evaluates on the test split
- Optionally runs inference on `testdata/`

## Environment and dependencies

This project is tested with **Python 3.11** and must be run inside a Python 3.11 virtual environment.

Create and activate a virtual environment, then install dependencies:

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

If PyTorch installation fails due to platform or CUDA differences, install the appropriate package from https://pytorch.org/ and then re-run `pip install -r requirements.txt`.

### Core dependencies

- `ultralytics`
- `torch`
- `opencv-python`
- `matplotlib`
- Grounding DINO dependencies

---

## Configuration notes

- `split_dataset.py` controls `TRAIN_RATIO`, `VAL_RATIO`, `TEST_RATIO`, and `SEED`.
- `train.py` contains training hyperparameters, model selection, and dataset paths.
- `auto_label.py` contains `BOX_THRESHOLD` and `TEXT_THRESHOLD` for Grounding DINO sensitivity.

> When you change class names or class count, update shared mappings and regenerate `yolo_dataset/data.yaml` by rerunning `split_dataset.py`.
